直接插入：

分成两块，前和后，前面是已经排好的序列

后面取第一个，在前面找比这个小的位置，找到了就在这个位置后面插入

希尔排序：

缩小增量排序

采用 分组 每组进行插入排序

按照增量跳着分组

越到后面增量越小

跳跃式的分组 可能会将 更小的元素 尽可能往前放

  1. 希尔排序是对直接插入排序的优化。
  2. 当gap > 1时都是预排序，目的是让数组更接近于有序。当gap == 1时，数组已经接近有序的了，这样就会很
    快。这样整体而言，可以达到优化的效果。我们实现后可以进行性能测试的对比。
  3. 希尔排序的时间复杂度不好计算，因为gap的取值方法很多，导致很难去计算，因此在好些树中给出的希尔排
    序的时间复杂度都不固定
  4. 不稳定，因为是跳跃式分组

选择排序：

思路：就是在后面选一个最小的元素与这个部分的第一个元素进行交换

时间复杂度分析：不管情况如何都是O(n^2)

空间复杂度:O(1)

稳定性：不是稳定的

还有一种比较高效的选择排序：就是定义两个指针left和right，再定义一个游标指针i，从left+1开始遍历到right-1（因为left和right都已经是有序的序列），在移动的过程中获取该段数组的最大值下标maxIndex和最小值下标minIndex,遍历完毕之后将left元素与minIndex元素交换，将right元素与maxIndex元素交换

堆排序：

用所给数组建一个大堆，然后数组的长度-1次将根节点和最后一个节点对调之后，每次都要从根节点向下调整

时间复杂度 n-1次 * 每次向下调整 O(logN)

所以最后堆排序的时间复杂度为O(n*logN)，所以 堆排序算是一种时间效率比较高的排序算法

Hoore法：

时间复杂度 最好是 O(n*logN)满二叉树/完全二叉树 最坏是 O(N^2) 正序/逆序 单分支树

空间复杂度 O(logN) 满二叉树/完全二叉树 最坏是 O(N) 单分支树

挖坑法：

​ 交替埋坑 找完之后就得到了基准 左边比基准小 右边比基准大

​ 先对基准值下标处挖坑（其实就是把这里的数据复制一份到tmp里）

​ right先走 找到比基准值小的位置 挖坑 把这里的数据挖出来填到上一个坑

​ 然后再让left走 找到比基准大的位置 挖坑 填到上一个坑

​ 再让right走 … 一直到left和right重合为止 此时 在此处挖坑 把数据填到上一个坑中 然后把tmp的值填到这个新挖的坑当中 就得到pivot = left = right了

左右指针法：

​ 定义一个前指针cur 一个后指针prev 定义cur比prev后一步 先判断cur是否小于key，然后对prev进行迭代操作，如果小于key且prev不等于cur那么就对prev和cur进行交换 每次循环对cur++操作 cur>right时退出循环 然后此时的prev就是pivot基准值所在位置 交换prev和key的位置 最后返回prev即可

栈的类型就是整型

当递归到只剩下一个元素的时候 要记得退出

根据数组的结构可以知道当pivot+1 == end时 右边只剩下一个元素 当pivot-1 == start时 左边只剩下一个元素

所以循环的条件应该是 pivot+1<end && pivot -1 > start

归并排序

一个序列 先不断分解成子序列 直到分解到一个个元素 再不断进行有序合并

往左边走 r = m

往右边走 l = m+1

三步：两边先分解（递归） ，最后合并

合并两个有序数组 ： 额外申请一整组空间 两个指针遍历两个数组 哪个小就放进去 如果其中一组已经走完 就一直放另外一组

然后按照下标顺序出队列重新放到数组中

计数排序

有一组 0～9 之间的数据

创建一个计数数组 对应可能出现的每一个数（按照顺序）

然后遍历原来的数组 统计每一个数出现的次数

最后按照计数数组结果输出 0不打印 其他则按照次数重复打印（重新写回当前数据）

申请多大？ 看数的范围 最大数 最小数 相差多少

怎么建立计数数组下标和可能出现的数字的关联 ？ 用当前的数值-最小值 就是对应计数数组的下标

重新写回 的时候 数组下标+最小值 恢复成原来的数据元素

时间复杂度：O（N+范围）

空间复杂度：O（范围）

时间效率最高的排序 但是也非常浪费空间 一定需要申请一个范围大小的空间

排序OJ分析

快速排序与其非递归实现：样例中出现了大量重复的数据，重复的数据用midOfThree方法也无法有效解决，所以不断开辟栈空间，二叉树深度不算加深 时间效率变成O(n^2)

归并排序 ，不受数据的顺序和是否重复影响 所以时间复杂度仍为O(n*logn)

堆排序：同样不受数组序列是否有序 重复影响 时间复杂度O(n*logn)

希尔排序：同样不受影响

基数排序因为样例中有负数出现造成报错

计数排序 因为是直接开辟一个超大的计数数组 且 没有比较直接输出 所以计数排序的时间复杂度是可以达到O(n+范围)的 几乎等于O(n) 所以时间效率是最高的
