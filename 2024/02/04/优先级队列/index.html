优先级队列

底层使用了堆 堆就是完全二叉树的基础上进行了一些调整

按完全二叉树的顺序存储方式存储 在一个一维数组中

堆的性质：
堆中某个节点的值总是不大于或不小于其父节点的值；
堆总是一棵完全二叉树。

根节点小于左右孩子 并且是一个完全二叉树 小根堆

根节点比左右孩子大 大根堆

数组中存储的是层序遍历的值

已知孩子节点下标 父节点下标为(i-1)/2

如果2 * i + 1 小于节点个数，则节点i的左孩子下标为2 * i + 1，否则没有左孩子
如果2 * i + 2 小于节点个数，则节点i的右孩子下标为2 * i + 2，否则没有右孩子

由一个初始数组创建大根堆：

从最后一棵树开始（这种树只有两层），向前面的节点进行遍历方法，先找到这棵树的根节点，再调用找到值更大的节点，与根节点的值进行比较，如果比根节点的值大，那么就调换位置，如此下去将p置于c的位置，再次迭代，直到找不到子节点（子节点的下标值大于数组的最大下标）

方法的参数（一个父节点用来定义一棵树，树的节点个数）

假设树的高度为h

​ T(n) = 2^0 (h-1) + 2^1(h-2) +….+2^(h-2)*1 结合思想来算时间复杂度

错位相减法最后得到 T(n) = 2^h-1- h

n = 2^h -1

h = log (n+1)

T(n) = n - log(n+1) = n –> T(n) = O(n)

当采用向下调整去建堆的时候 时间复杂度为O(n)

插入操作

  1. 80往哪里放
  2. 放进去怎么放到合适的位置 –> 向上调整

child == 0时结束 找到父节点 p p与c不断变换迭代

删除操作

优先级高删除 那么就是大根堆的顶部 也就是数组的第一个元素

方法思路：先交换头下标和最后一个下标，然后再对根进行调整（向下调整）

PriorityQueue:

多个构造方法：

不带参数

指定容量

指定比较器

容量+比较器

queue: Object类型数组

默认容量：11

用一个集合创建优先级队列（传入实现collection接口的参数）

扩容方法：

​ 如果在64以下，+2

​ 64以上 二倍扩容

如果超过MAX__ARRAY_SIZE 就按照其他的方法扩容

Oj:

topK问题：求数据集合中钱K个最大的元素或者最小的元素，一般情况下数据量都比较大

采用 PriorityQueue 19ms

用选择排序 直接超时

做法1:把数组排序 排序之后取出钱10个最大的 数据里那个非常大的时候 你无法在内存中排序

做法2:优先级队列 出队k次 效率不高 复杂度为O(nlogn + klogn) = O((n+k)logn) 因为n比k大，所以logn一定比logk大 这种做法明显复杂度是要高的

建议的做法：（用复杂度或者思路去分析耗时应该是更小的）

前K个最大的数据 先建立 小根堆 K个元素（直接数组前K个元素）

小根堆最上面的元素是最小的元素 拿数组中的元素与堆顶元素比较 如果比堆顶元素小 那么可以知道数组中的这个元素一定不是最大的k个元素之一

(O(K)+O(N-K))*logK(调整)

最后的复杂度是O(n*logK)

时间最小的还是快速排序

找第K大的元素 小根堆K个元素 找第K小的元素元素 大根堆K个元素 最后都是把堆顶元素取出来就可以了

堆的应用–利用堆作为底层结构封装优先级队列

向上调整 时间复杂度O(nlogn)

向下调整 时间复杂度O(logn)

堆排序：从小到大进行排序

所有的排序都指的是对原来的数据进行排序

建立一个大根堆，每次把堆顶 和 堆的最后一个元素进行交换 这样最大根就是有序的（也就是到了它应在的位置）

每次换位之后要进行调整

相当于是从数组的最后一个元素开始进行排序

降序排序也差不多 只不过换成了小根堆
