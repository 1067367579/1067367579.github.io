OJ题：

找到链表的中间节点，奇数个就返回中间那个，偶数个返回中间两个的后面一个，要求只能遍历一遍实现

思路：双指针，一个fast一个slow，fast每次走两步，slow每次走一步，走到最后slow就是到了中间节点，（对于奇偶都适用）

原理：速度*时间 = 距离（速度两倍，距离自然也两倍）

Fast = fast.next.next

slow = slow.next

循环条件：fast!=null && fast.next!=null 需注意条件的顺序不可变化（利用短路规避空指针问题）

找倒数第k个节点：5-2 = 3

可以直接减k得步数，也可以像上面一样利用它fast和slow指针

fast要领先slow k-1步，然后同时走，fast到末尾时slow就到目标节点

原理：速度一致，两个指针之间的距离不变（路程差确定）

合并两个有序链表变成一个链表

首先定义一个newHead指针和一个tmpHead指针，一个表示合成后链表的头节点，一个表示当前合成链表的尾节点

然后遍历两个链表，比较两个链表当前指针指向元素的大小，小的一个使tmpHead指针的next域指向它，然后更新tmpHead和这个链表的当前指针

最后其中一个链表到尾之后结束循环

因为两个链表的长度不一定相等，所以要检查是哪个链表到尾巴了，没有到尾巴的那一个使tmpHead的next域指向它

最后返回头节点的next即可（因为头节点无意义）

将链表按标识大小分成两块

分成两块区域，并且还需要把这两块区域连接起来重新组成一个链表

所以我们要维护两块区域，before和after区域，用四个指针bs,be,as,ae维护

首先bs,be,as,ae都是空指针，用于标识是否插入了，因为两块区域第一次插入的情况和后面插入的情况都是不同的（无头节点链表的特性）

循环，定义cur指针遍历原链表

比较cur数据域与标识大小，小就插入before区域，大就插入after区域

第一次插入时，bs,be指针都指向cur，此时be无需移动

后面的插入，be指针的next指向cur，尾插法连接，然后be指向下一个节点

结束循环之后，因为无法确保before和after的情况，需要判断空指针的问题

即要检查before为空时就没有连接这一步，直接返回as

检查as不为空时，这是要把ae的next指针置为空，不然变成循环链表，遍历时会产生死循环的问题

然后才可连接be，as，连接之后返回bs即可

比较，1.没有相遇（循环条件）

链表的公共点：

首先，两个链表的相交情况一定是Y字形，不可能是X字形，因为链表的节点只有一个next域，不会有两个next域

不一样长的地方一定是相遇前的两段，后面一定是一样的（Y字形）

可以先让 最长的链表的引用 先走他们的差值步 再以同样的速度往后走使其相遇

判断一个链表是否有环

还是一样的思想，一个快指针，一个慢指针，如果这两个指针同时出发，会再次相遇，则这说明这个链表有环

原理：速度不一样起点相同还能相遇，只能说明是在一个环内运动

规则：快的速度是慢的速度两倍，最快相遇，3倍4倍可能会出现不会相遇的情况（节点个数不同时）

一个走一步。一个走两步，追及问题

顺序表

顺序表

字段：

​ 底层结构：一个数组

​ 实际顺序表大小：size

​ 数组的容量：capacity

​ 默认容量（常量）:DEFAULT_CAPACITY

Add 方法：

  • 返回值：void

  • 接收参数：尾插法（只有一个参数data）,指定位置插入（有位置参数pos，数据参数data）

  • 要完成的工作：

    1. 指定位置插入，要判断pos位置的合法性，<0或者>size都是不可以的，可以抛出异常（自定义要重载信息）
1. 判断此时底层数组是否已满，如果满的话就要对数组进行扩容（搬到一个两倍大的数组里面）
1. 尾插法直接在size位置放入数据，指定位置插入则要从后往前一次往后挪动一个位置，直到指定的pos位置，挪动后插入到pos位置
1. 对size进行更新

Contains方法：

  • 返回值：boolean
  • 接收参数：要查找的对象
  • 循环遍历顺序表，长度为size，因为是引用类型，所以要用equals方法进行比较，如果相等就返回true
  • 最后返回false

indexOf方法：

  • 返回值：Integer
  • 接收参数：要查找的对象
  • 剩余和上面Contains方法基本相同，找到区别是返回下标值，找不到就返回-1

get方法：

  • 返回值：元素类型
  • 接收参数：位置参数pos
  • 判断pos的合法性，pos<0或者pos>=size直接抛出异常
  • 然后直接返回对应位置的元素即可

set方法（更新）

  • 返回值：void
  • 接收参数：位置参数pos,值value
  • 和上面一样判断pos合法性
  • 然后直接设置即可

getSize方法：

直接返回size

remove方法：

  • 根据值通过indexOf方法找到位置，找不到就直接抛异常

  • 找到了从pos开始，到size-2,（i= pos;i<size-1;i++）用[i] = [i+1]的方式进行值的覆盖，然后使size-1即可

  • 最后就直接使size-1

  • 引用类型要使空出来的之前最后一个元素置为空

clear方法：

  • 引用类型要把之前的每一个位置都置空

  • 直接让size为0

Notes：

  • 用某个接口类型构造对象，就只能调用本接口的对应方法

  • ？extends E : 是E的子类或者E本身

  • 当调用不带参数的构造方法的时候，默认在第一次add的时候才会分配大小为10的内存，扩容是按照1.5倍的方式扩容

  • Java里面涉及到范围的地方一般都是左闭右开

  • subList并没有真正地把指定范围的数据拷贝出来，而是直接引用，所以更新会出现一些异常情况

遍历方式：

  • 直接sout，内部的toString方法
  • fori循环，get
  • 增强for
  • 迭代器遍历（迭代器还可以从后往前遍历，hasprevious(),构造时传入最后位置sIze）

排序

比较排序

排序讲究的速度是快，要以最快的速度按照指定的顺序排好

排序的稳定性问题：

内部排序：在内存上的排序

外部排序：数据量太大内存无法满足需要，将全部数据迁移到磁盘中进行排序

常见的排序算法：

插入排序：直接插入排序，希尔排序

选择排序：选择排序，堆排序

直接插入：

分成两块，前和后，前面是已经排好的序列

后面取第一个，在前面找比这个小的位置，找到了就在这个位置后面插入

希尔排序：

缩小增量排序

采用 分组 每组进行插入排序

按照增量跳着分组

越到后面增量越小

跳跃式的分组 可能会将 更小的元素 尽可能往前放

  1. 希尔排序是对直接插入排序的优化。
  2. 当gap > 1时都是预排序，目的是让数组更接近于有序。当gap == 1时，数组已经接近有序的了，这样就会很
    快。这样整体而言，可以达到优化的效果。我们实现后可以进行性能测试的对比。
  3. 希尔排序的时间复杂度不好计算，因为gap的取值方法很多，导致很难去计算，因此在好些树中给出的希尔排
    序的时间复杂度都不固定
  4. 不稳定，因为是跳跃式分组

选择排序：

思路：就是在后面选一个最小的元素与这个部分的第一个元素进行交换

时间复杂度分析：不管情况如何都是O(n^2)

空间复杂度:O(1)

稳定性：不是稳定的

还有一种比较高效的选择排序：就是定义两个指针left和right，再定义一个游标指针i，从left+1开始遍历到right-1（因为left和right都已经是有序的序列），在移动的过程中获取该段数组的最大值下标maxIndex和最小值下标minIndex,遍历完毕之后将left元素与minIndex元素交换，将right元素与maxIndex元素交换

堆排序：

用所给数组建一个大堆，然后数组的长度-1次将根节点和最后一个节点对调之后，每次都要从根节点向下调整

时间复杂度 n-1次 * 每次向下调整 O(logN)

所以最后堆排序的时间复杂度为O(n*logN)，所以 堆排序算是一种时间效率比较高的排序算法

快速排序：

Hoore法：

时间复杂度 最好是 O(n*logN)满二叉树/完全二叉树 最坏是 O(N^2) 正序/逆序 单分支树

空间复杂度 O(logN) 满二叉树/完全二叉树 最坏是 O(N) 单分支树

挖坑法：

​ 交替埋坑 找完之后就得到了基准 左边比基准小 右边比基准大

​ 先对基准值下标处挖坑（其实就是把这里的数据复制一份到tmp里）

​ right先走 找到比基准值小的位置 挖坑 把这里的数据挖出来填到上一个坑

​ 然后再让left走 找到比基准大的位置 挖坑 填到上一个坑

​ 再让right走 … 一直到left和right重合为止 此时 在此处挖坑 把数据填到上一个坑中 然后把tmp的值填到这个新挖的坑当中 就得到pivot = left = right了

左右指针法：

​ 定义一个前指针cur 一个后指针prev 定义cur比prev后一步 先判断cur是否小于key，然后对prev进行迭代操作，如果小于key且prev不等于cur那么就对prev和cur进行交换 每次循环对cur++操作 cur>right时退出循环 然后此时的prev就是pivot基准值所在位置 交换prev和key的位置 最后返回prev即可

归并排序

一个序列 先不断分解成子序列 直到分解到一个个元素 再不断进行有序合并

往右边走 l = m+1

合并两个有序数组 ： 额外申请一整组空间 两个指针遍历两个数组 哪个小就放进去 如果其中一组已经走完 就一直放另外一组

计数排序

有一组 0～9 之间的数据

创建一个计数数组 对应可能出现的每一个数（按照顺序）

然后遍历原来的数组 统计每一个数出现的次数

最后按照计数数组结果输出 0不打印 其他则按照次数重复打印（重新写回当前数据）

申请多大？ 看数的范围 最大数 最小数 相差多少

重新写回 的时候 数组下标+最小值 恢复成原来的数据元素

排序OJ分析

快速排序与其非递归实现：样例中出现了大量重复的数据，重复的数据用midOfThree方法也无法有效解决，所以不断开辟栈空间，二叉树深度不算加深 时间效率变成O(n^2)

归并排序 ，不受数据的顺序和是否重复影响 所以时间复杂度仍为O(n*logn)

堆排序：同样不受数组序列是否有序 重复影响 时间复杂度O(n*logn)

希尔排序：同样不受影响

基数排序因为样例中有负数出现造成报错

计数排序 因为是直接开辟一个超大的计数数组 且 没有比较直接输出 所以计数排序的时间复杂度是可以达到O(n+范围)的 几乎等于O(n) 所以时间效率是最高的
